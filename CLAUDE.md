## Build Commands

```bash
./gradlew assembleDebug          # Debug APK
./gradlew assembleRelease       # Release APK (set VERSION_NAME property)
./gradlew testDebugUnitTest     # Unit tests
./gradlew lintDebug             # Lint checks
./gradlew jacocoTestReport      # Coverage report (output: app/build/reports/jacoco/)
```

## Architecture

**Clean Architecture (MVVM)** with three layers:
- `domain/` - Models, repository interfaces, use cases
- `data/` - Room database, repository implementations, CSV import
- `ui/` - Compose screens, ViewModels, navigation

**Database**: Single `StudyDatabase` (Room) with 8 DAOs: Session, Flashcard, Subject, Grade, Question, PracticeRecord, ImportRecord, WrongAnswerBook

**Navigation**: Jetpack Navigation Compose with sealed `Screen` class. Routes pass parameters (subjectId, gradeId, questionCount, timeLimitSeconds) via NavType.

**User Flows**:
1. **Child mode**: Home → SubjectGradeSelect → Settings → Quiz → Result
2. **Parent mode**: Home → ParentHome → (Device lock screen authentication, Subject/Grade management, CSV import, History, Import records)

## Key Patterns

- **Dependency Injection**: Hilt modules in `di/` (DatabaseModule, RepositoryModule)
- **CSV Import**: `CsvImporter.parseLine()` handles CHOICE and FILL_BLANK question types
- **Repository Pattern**: Domain repository interfaces in `domain/repository/`, implementations in `data/repository/`
- **Entity Mapping**: Room entities in `data/local/entity/`, domain models in `domain/model/`

## CI/CD

- `android-ci.yml`: Runs on `main` and `dev*` branches - unit tests, lint, debug APK, JaCoCo coverage, GitHub Pages deployment
- `android-release.yml`: Runs on `main` push and `v*` tags - release APK, GitHub Release creation
- **GitHub Pages**: Test reports at `https://yiranlaux.github.io/my-study/jacoco/jacocoTestReport/html/`
- **Helper script**: `./android-ci.sh -b` builds via CI and downloads to `build/my-study-debug.apk` (skips copy if APK already exists, improved download robustness)

## Testing

Unit tests use Mockito + Kotlin Coroutines test. Repository tests mock DAOs. ViewModel tests use `runTest`. Test coverage reports generated by JaCoCo.
